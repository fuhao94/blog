## Day1：React Fiber 架构 & 调和流程

- 学习内容：Fiber链表、双缓存、Diff算法

- 面试题：
    1. Fiber相比Stack Reconciler的核心优势是什么？
    2. React Diff算法的三个优化策略是什么？

### Fiber相比Stack Reconciler的核心优势是什么？

* 可中断的异步渲染

将同步递归的调和过程拆解为可暂停/恢复的异步任务

* 优先级调度

高优先级的人物可以打断低优先级任务（如用户输入打断数据更新）

* 并发模式基础

Concurrent Mode 基础；Suspense useTransition 等特性提供底层支持

**总结：** Fiber 把 React 的同步递归更新，改成了可中断、可恢复、可优先级调度的异步任务，解决了 Stack Reconciler 卡顿掉帧的根本问题。


### React Diff算法的三个优化策略是什么？

React Diff 之所以快，是因为`只同级比`、`不同类型直接重建`、`同级靠 key 精准复用`，用三个符合真实页面场景的策略，把复杂的 Diff 变得极简高效。

总结：
`同层比较`：不跨层级 Diff，减少计算量
`类型不同则重建`：快速淘汰不可复用节点
`key 标识复用`：列表元素精准匹配，提升性能


### Fiber 局限性

* 比传统虚拟DOM更占内存
* 学习成本相对较高

### Fiber 协调流程

> Fiber 协调 = 调度优先级 + 可中断计算（Render） + 同步提交 DOM（Commit）

先算好要改什么，再一口气改到页面上，计算过程还能随时给浏览器让路。

1. 触发更新
2. Scheduler 调度
3. render 阶段
4. commit 阶段
5. 收尾（重置指针 准备下一次更新）

### Fiber 工作全流程

1. 更新触发：setState、props 变化、forceUpdate 等。
2. 调度优先级：Scheduler 分配优先级，决定何时执行。
3. Render 阶段： 
   * 从 root 开始深度遍历 Fiber 树。
   * beginWork：Diff、创建新 Fiber、标记 Effect。
   * completeWork：收集 Effect 到 Effect List。
   * 可中断、可恢复。
4. Commit 阶段： 
   * 遍历 Effect List，操作 DOM。 
   * 执行生命周期、hooks 回调。
5. 完成，视图更新。


## Day2：Hooks 原理与闭包陷阱

- 学习内容：Hooks链表结构、useState/useEffect实现、依赖项本质

### hooks 链表结构

函数组件是纯函数，执行完就销毁，内部变量会被清空。

维护了一条专属链表：按照hooks调用顺序串成的单向链表，用来记录和复用每个hooks的状态，保证渲染稳定

### 为什么Hooks不能在条件/循环中使用？

状态靠链表顺序匹配，所以必须写在顶层

### useEffect闭包陷阱的本质与解决方案？

useEffect 在某次渲染时创建，捕获了当时的变量快照，之后永远使用这份旧快照，不会感知新渲染的变量

`useEffect 闭包陷阱不是 bug，是 JS 闭包 + React 渲染模型的自然结果，补上依赖或用 useRef 存最新值就能彻底解决。`

## Day3：React 并发特性 & Scheduler

### 并发特性

React 可以 中断、暂停、恢复、放弃渲染过程，优先保证用户交互和页面流畅，不被长任务阻塞

原理：
1. 并发渲染器：任务切片
2. 时间切片：每片只执行很短时间
3. 优先级调度

### Scheduler

是 React 内置的优先级调度器，基于时间切片(time slicing)实现，负责给react更新任务分配的优先级，保证页面永远流畅。

核心作用
1. 任务的优先级管理

高低优先级

2. 时间切片

把长渲染拆分成小任务片，主动让出主线程，避免阻塞浏览器渲染和交互

3. 协调并发渲染

提供并发特性的底层支持
